# Atividade Substitutiva · Global Solution

## 2TWDOR

## Plataforma Integrada de Comunicação e Coordenação

**Atividade Individual**

---

## Contexto

Esta atividade aborda as competências técnicas exigidas em **Global Solution**:

- Desenvolvimento full stack;
- Integração entre app mobile e interface web;
- Backend funcional;
- Deploy completo.

O tema é aberto, de livre escolha, e deve ser desenvolvido de forma individual.

Esta atividade substitutiva parte do mesmo conjunto de habilidades exigidas no projeto desenvolvido na Global Solution:

- Concepção de plataforma;
- Design visual;
- Navegação;
- Interatividade;
- Publicação online.

A proposta, agora, permite um tema livre, desde que a solução seja desenvolvida individualmente.

---

## O Desafio

Você deve projetar e publicar uma plataforma integrada composta por:

- App mobile;
- Interface web;
- Backend com API REST.

O contexto e a aplicação são de livre escolha. O importante é que a solução faça sentido como produto real, com propósito claro e arquitetura tecnicamente coerente.

---

## Sugestões de Tema

O tema é livre, mas seguem algumas sugestões de ponto de partida:

- Plataforma de gestão e comunicação para equipes em campo, como obras, eventos ou operações logísticas;
- App de monitoramento e registro para comunidades rurais ou agricultores;
- Plataforma de comunicação assíncrona para equipes remotas e distribuídas;
- Sistema de gestão de tarefas e status para pequenas clínicas, ONGs ou cooperativas.

Essas são apenas inspirações. É permitido propor algo diferente, desde que a integração entre app mobile e interface web esteja presente e a arquitetura técnica seja funcional e documentada.

---

## Entrega Esperada

A atividade é composta por **2 itens**.

---

## 1. Documento PDF

O documento PDF deve conter:

- RM e nome completo do aluno;
- Descrição da plataforma proposta:
  - O que é;
  - Para quem serve;
  - Qual problema resolve;
- Justificativa da escolha do tema;
- Justificativa das escolhas de design e experiência do usuário:
  - Identidade visual;
  - Paleta de cores;
  - Tipografia;
  - Navegação;
- Descrição da arquitetura técnica da solução:
  - Como o app se integra ao backend;
  - Como a interface web se integra ao backend;
  - Como o backend disponibiliza dados por meio da API REST;
- Link do repositório público no GitHub;
- Link do deploy.

---

## 2. Plataforma com Repositório no GitHub

Deve ser entregue uma plataforma integrada composta por:

- App mobile;
- Interface web;
- Backend funcional com API REST;
- Deploy completo;
- Repositório público no GitHub.

O conteúdo da plataforma pode ser fictício, mas a integração entre app mobile, interface web e backend deve ser real e demonstrável.

---

## Critérios de Avaliação

A atividade será avaliada com base nos seguintes critérios:

- Clareza e criatividade na proposta da plataforma;
- Qualidade da integração entre app mobile e interface web;
- Usabilidade e experiência do usuário em ambas as plataformas;
- Robustez do backend e coerência da arquitetura técnica;
- Deploy completo e funcional;
- Recursos de acessibilidade.

---

## Observações

- A atividade é estritamente individual;
- Não serão aceitos projetos já desenvolvidos em outras disciplinas ou desafios do curso;
- O repositório no GitHub deve ser público;
- O repositório deve conter todos os arquivos do projeto:
  - App mobile;
  - Interface web;
  - Backend.
